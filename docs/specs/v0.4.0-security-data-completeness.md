## Problem

Security specialist and AI Harness specialist found hardening gaps and data completeness issues:

**Security:**
- Adapters follow symlinks â€” could detect/load files outside project root
- `_read_bounded` missing `encoding="utf-8"` in session-schema adapter
- Redaction misses `AWS_SECRET_ACCESS_KEY=...` and JWTs split across content blocks
- `load_json_report` reads without size limit â€” potential OOM
- `_validate_judge_log_path` uses fragile `Path.cwd()` as security boundary

**Ragas:**
- `temperature` from MetricConfig is never passed to `llm_factory()`
- Faithfulness/relevancy import from private Ragas modules (`_faithfulness`, `_answer_relevance`)
- `embedding_factory()` assumes OpenAI defaults, inconsistent with Vertex AI setup

**Adapter completeness:**
- `model_id` never populated by session-schema adapter
- `tokens_in`/`tokens_out` never populated by session-schema adapter
- `DatasetLoader` is non-recursive â€” nested session dirs silently produce zero results
- Generational file sorting assigns wrong generation to base file
