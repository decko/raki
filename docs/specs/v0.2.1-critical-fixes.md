# v0.2.1 — Critical Fixes

**Date**: 2026-04-19
**Status**: Ready
**Priority**: Immediate — v0.2.0 has 2 runtime-breaking bugs

## Problem

The v0.2.0 code review by 6 specialists found 2 critical bugs:

1. **Python 2 `except` syntax** in `session_schema.py` lines 126 and 152. `except KeyError, ValueError:` is a `SyntaxError` in Python 3 — it should be `except (KeyError, ValueError):`. The session-schema adapter's error recovery branches in `_load_findings()` and `_load_events()` are completely broken. Tests pass only because fixtures never trigger those branches.

2. **Ragas `ascore()` called with kwargs** in `precision.py` and `recall.py`. Ragas 0.4 `ascore()` expects a `SingleTurnSample` object, not keyword arguments. This will `TypeError` at runtime with real Ragas. Tests pass because mocks accept anything. Faithfulness and relevancy metrics correctly use `SingleTurnSample` — only precision/recall are broken.

Additionally:
3. **`--json` stdout corruption** — `console.print()` writes Rich output to stdout, mixing with JSON when `--json` is used. `--json` should imply `--quiet`, or console should write to stderr.
4. **Version still says 0.1.0** in pyproject.toml and `__init__.py`.

## Tasks

### Task 1: Fix except syntax + version bump

**Files**: `src/raki/adapters/session_schema.py`, `pyproject.toml`, `src/raki/__init__.py`

- Fix `except KeyError, ValueError:` → `except (KeyError, ValueError):` on lines 126 and 152
- Add test cases that trigger the except branches (malformed finding missing `issue` key, malformed event line)
- Bump version to `0.2.1` in both pyproject.toml and `__init__.py`

### Task 2: Fix Ragas ascore() to use SingleTurnSample

**Files**: `src/raki/metrics/ragas/precision.py`, `src/raki/metrics/ragas/recall.py`

- Construct `SingleTurnSample` from row data before calling `ascore()`, matching the pattern in faithfulness.py/relevancy.py
- Use `single_turn_ascore()` consistently across all 4 Ragas metrics (not `ascore()` on some and `single_turn_ascore()` on others)
- Import from public `ragas.metrics.collections` consistently (not private `ragas.metrics._faithfulness`)
- Update tests to verify `SingleTurnSample` construction

### Task 3: Fix --json stdout corruption

**Files**: `src/raki/cli.py`

- When `--json` flag is set, write Rich output to stderr via `Console(stderr=True)` or imply `--quiet`
- Add test: `raki run --json -m manifest.yaml --no-llm | python -m json.tool` should produce valid JSON
