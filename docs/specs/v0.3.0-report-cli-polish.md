## Problem

The v0.2.0 HTML report and CLI have usability issues identified by UXD, CLI, and Python specialists:

- HTML report shows wrong colors for inverted metrics (rework shows red when low is good)
- Sessions count shows 0 because `sample_results` is never populated
- Raw snake_case metric names shown instead of human-readable labels
- Empty states not handled â€” sections vanish silently with no message
- No metric descriptions or interpretive guidance
- CLI `--quiet` doesn't suppress auto-discovery in validate
- No test for threshold exit-code-1
- `adapters` command has no descriptions
- 4 soda sessions crash with NoneType error
- Accessibility gaps in HTML (color contrast, ARIA labels)
